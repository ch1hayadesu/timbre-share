## 需求描述

### 背景

- **业务问题**：
  - **音色获取困难**：内容创作者面临音色获取困难、成本高昂的痛点。专业配音费用高、沟通周期长，而现有语音克隆工具操作复杂、技术门槛高，普通用户难以使用。市场上缺乏一个低门槛、一站式的音色克隆与合成平台。
  - **内容创作效率低**：用户制作多人对话内容（如有声短剧、配音作品）时，需要手动为每个角色分别选择音色、逐句输入文本进行合成，流程繁琐、耗时长。
  - **音色复用困难**：用户克隆的优质音色无法被其他用户发现和使用，每个用户都需要从零开始克隆，音色资源无法流通共享，平台生态封闭。

- **目标用户**：
  - 内容创作者（短视频、有声书、播客制作者）
  - 自媒体运营者
  - 中小企业（广告配音、培训课件）
  - AI短剧创作者
  - 有声内容制作者
  - 音色分享者与使用者
  - 有个人声音数字化需求的普通用户

- **价值主张**：
  - **基础平台**：通过「上传即克隆」的低门槛体验，让每个用户都能快速将自己的声音数字化，并随时通过文本驱动生成自然流畅的语音内容。
  - **语音智能应用**：将「逐句手动合成」升级为「上传剧本→一键生成完整配音」，大幅提升内容创作效率。
  - **分享平台**：打通音色流通壁垒，构建「克隆→分享→使用」的音色生态闭环。

### 功能概述

#### 模块一：语音克隆与TTS合成（基础平台）

- **核心功能**：
  1. 语音克隆：用户上传任意音频 → 系统自动降噪预处理 → 即时克隆（快速预览）/ 深度克隆（高精度训练）
  2. 文本转语音（TTS）：输入文本 → 选择已克隆的音色 → 调节语速/音量/音调 → 合成 MP3 音频

- **功能边界**：
  - MVP 包含：语音克隆（即时+深度）、TTS 合成（基础参数）、音色管理（增删查）
  - MVP 不包含：多语言合成、智能音频编辑、多轨道混音

#### 模块二：语音智能应用（剧本智能配音）

- **核心功能**：
  1. 用户上传txt格式剧本文件
  2. 系统自动识别并分割角色
  3. 对每句台词进行情感分析（喜悦/愤怒/悲伤/恐惧/厌恶/中性）
  4. 根据情感自动调节音调、语速等参数
  5. 从用户语音库中智能匹配最合适的音色
  6. 一键生成完整MP3配音音频
  7. 生成后用户可对结果进行修改调整

- **功能边界**：
  - 包括：txt剧本上传、角色分割、情感分析、音色匹配、整段合成、结果修改
  - 不包括：处理过程中的预览与中途调整（MVP阶段）、自定义角色音色映射（MVP阶段）
  - MVP阶段限定：3-4个角色的小型AI短剧，每角色台词不超过50句

#### 模块三：分享平台（音色模型分享）

- **核心功能**：
  1. 用户将自己克隆的音色模型发布到平台公共区
  2. 其他用户可浏览、搜索平台上的共享音色
  3. 其他用户可下载共享音色到自己的语音库
  4. 下载后的音色可直接用于TTS合成和剧本智能配音

- **功能边界**：
  - 包括：音色发布/下架、音色浏览与搜索、音色下载
  - 不包括：音色审核机制（MVP阶段）、音色描述标签（MVP阶段）、下载次数限制（MVP阶段）、音色定价/交易（后续版本）

### 用户场景

#### 场景一：语音克隆与TTS合成
> 短视频创作者上传自己的一段录音 → 克隆出个人音色 → 输入脚本文案 → 生成配音 MP3 → 用于视频制作

#### 场景二：教育内容制作
> 教育内容制作者上传讲师的音频 → 克隆讲师音色 → 批量合成课件语音 → 用于在线课程

#### 场景三：个人趣味体验
> 普通用户出于好奇上传自己的声音 → 体验克隆效果 → 用克隆音色生成趣味语音分享

#### 场景四：剧本智能配音
> 小王是一名AI短剧创作者，他写了一个3个角色的短剧剧本（txt格式），想快速生成配音音频。他登录音色共享平台，进入「剧本配音」页面，上传剧本文件。系统自动识别出3个角色、分割台词、分析情感，从小王已有的语音库中为每个角色匹配合适的音色，然后生成一段完整的MP3配音音频。小王试听后对某个角色的音色不满意，手动更换了音色后重新生成，最终满意地下载了成品。

#### 场景五：音色模型分享
> 小李克隆了一个非常逼真的"温柔女声"音色，觉得效果很好，想分享给其他用户。他进入音色管理页面，点击「分享到平台」，音色即出现在平台的公共音色区。小张在浏览公共音色区时搜索到该音色，试听后觉得效果不错，点击下载将其添加到自己的语音库中，之后在自己的TTS合成和剧本配音中使用该音色。

## 详细需求

### 模块一：语音克隆与TTS合成

#### 1. 用户系统

| 项目 | 说明 |
|------|------|
| 注册方式 | 手机号 + 短信验证码 |
| 登录方式 | 手机号 + 短信验证码 |
| 用户身份 | 所有注册用户功能一致（MVP 不区分免费/付费） |

#### 2. 语音克隆模块

| 项目 | 说明 |
|------|------|
| 音频上传 | 支持 MP3、WAV、M4A、OGG 等常见格式，文件大小 ≤50MB |
| 预处理 | 系统自动降噪、去除背景音、音量归一化，无需用户手动操作 |
| 即时克隆模式 | 上传后 1-3 分钟内生成可用的克隆音色，供快速预览和轻量使用 |
| 深度克隆模式 | 后台异步训练（预计 10-30 分钟），完成后站内通知 + 可选短信通知，生成高精度音色 |
| 音色命名 | 用户可为克隆的音色自定义名称 |
| 音色试听 | 每个克隆音色提供预设文本的试听功能 |
| 音色删除 | 用户可随时删除自己克隆的音色 |

#### 3. TTS 合成模块

| 项目 | 说明 |
|------|------|
| 文本输入 | 单次 ≤1000 字 |
| 音色选择 | 从用户已克隆的音色列表中选择 |
| 参数调节 | 语速（0.5x - 2.0x）、音量（0-100%）、音调（-12 ~ +12 半音） |
| 合成输出 | MP3 格式音频文件，支持在线播放和下载 |
| 合成历史 | 记录历史合成记录，支持重新生成和再次下载 |

#### 4. 输入/输出规格

| 模块 | 输入 | 输出 |
|------|------|------|
| 语音克隆 | 用户音频文件（≤50MB，常见格式） | 音色模型（即时版 / 深度版） |
| TTS 合成 | 文本（≤1000字）+ 音色选择 + 参数设置 | MP3 音频文件 |
| 用户注册 | 手机号 + 验证码 | 用户账户 |

#### 5. 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 上传音频过短（<5秒） | 提示用户上传至少 5 秒的音频样本 |
| 上传音频纯噪音/无人声 | 预处理阶段检测并提示「未检测到有效人声，请重新上传」 |
| 上传文件过大（>50MB） | 前端限制 + 后端校验，提示文件大小超限 |
| 克隆训练失败 | 显示失败状态，提供「重新克隆」按钮 |
| TTS 文本为空 | 前端禁用合成按钮，提示输入文本 |
| TTS 文本超限（>1000字） | 前端实时字数统计，超限时截断并提示 |
| 无可用音色时合成 | 提示用户「请先克隆一个音色」并引导至克隆页面 |
| 并发克隆 | 同一用户同时只能进行 1 个深度克隆任务，新任务排队等待 |
| 网络中断/超时 | 合成中断时提示用户重试，不扣减额度 |

### 模块二：语音智能应用（剧本智能配音）

#### 1. 功能介绍

剧本智能配音模块允许用户上传txt格式剧本文件，系统自动完成角色分割、情感分析、音色匹配，一键生成完整的多人配音MP3音频。MVP阶段采用"黑盒处理"模式，用户上传剧本后等待生成结果，生成后可在结果页进行调整。

**核心功能列表**：
1. 支持上传txt格式剧本文件
2. 智能识别并分割剧本中的角色
3. 对每句台词进行情感分析（喜悦/愤怒/悲伤/恐惧/厌恶/中性）
4. 根据情感自动调节TTS参数（音调、语速）
5. 从用户语音库中智能匹配各角色音色
6. 一键生成完整MP3配音音频
7. 生成结果支持试听、修改调整、下载

#### 2. 输入

| 输入项 | 描述 |
|--------|------|
| **输入来源** | 用户通过Web前端上传txt格式剧本文件 |
| **文件格式** | 仅支持 .txt 文本文件 |
| **文件大小** | ≤5MB |
| **角色数量** | 3-4个角色（MVP阶段），超过3个角色弹窗提示并停止 |
| **台词规模** | 每角色台词不超过50句 |
| **编码格式** | 支持UTF-8、GBK编码自动识别 |

**剧本格式示例**：
```
角色A：你好，今天天气真好。
角色B：是啊，我们出去走走吧。
角色A：好的，我马上就来。
（角色C匆匆跑来）
角色C：等等我！我也要去！
```

#### 3. 处理流程

```
用户上传txt剧本文件
    ↓
文件格式与大小校验
    ↓
文本编码识别（UTF-8/GBK）
    ↓
剧本结构解析 → 角色识别与分割
    ↓
角色数量检查 → 超过3个角色？→ 弹窗提示「剧本角色过多，当前仅支持3个角色以内的小型剧本」，停止处理
    ↓ （未超过）
逐句情感分析（喜悦/愤怒/悲伤/恐惧/厌恶/中性）
    ↓
从用户语音库匹配各角色音色
    ↓
用户语音库为空？→ 弹窗提示「还没有任何声音哦，请先去挑选声音，再来试试吧」，引导至音色市场/克隆页面
    ↓ （语音库有音色）
情感参数映射 → 自动调节音调/语速
    ↓
逐句TTS合成（多角色音频拼接）
    ↓
生成完整MP3配音音频
    ↓
返回结果页：试听播放器 + 音频下载 + 修改调整入口
```

**处理规格**：

A. **剧本解析**：
   - 使用NLP技术识别角色标识（如"角色名："、"角色名说："等模式）
   - 支持旁白/描述文本的识别与过滤
   - 若无法识别为剧本格式，默认将全文作为中性独白朗读

B. **情感分析**：
   - 对每句台词进行情感分类，输出以下6种情感之一：
     - 喜悦：积极、开心、兴奋的表达
     - 愤怒：生气、不满、激动的表达
     - 悲伤：难过、失落、忧郁的表达
     - 恐惧：害怕、紧张、惊恐的表达
     - 厌恶：反感、嫌弃、不屑的表达
     - 中性：陈述、说明、无明显情感倾向的表达
   - 无法判断情感的台词默认归为"中性"

C. **情感→参数映射规则**：

| 情感 | 音调偏移 | 语速倍率 | 其他 |
|------|----------|----------|------|
| 喜悦 | +2~+4 半音 | 1.1x~1.3x | 音色明亮 |
| 愤怒 | +1~+3 半音 | 1.2x~1.5x | 音量稍高 |
| 悲伤 | -2~-4 半音 | 0.7x~0.9x | 音色低沉 |
| 恐惧 | -1~-3 半音 | 1.0x~1.3x | 略带颤抖 |
| 厌恶 | -1~+1 半音 | 0.9x~1.1x | 语气冷淡 |
| 中性 | 0 半音 | 1.0x | 默认参数 |

D. **音色匹配逻辑**：
   - 匹配来源：用户个人语音库（包含：系统预设音色 + 用户克隆音色 + 用户从平台下载的音色）
   - 匹配策略：
     1. 若用户提供了角色特征描述（如"年轻女性"），按特征匹配语音库中相应音色
     2. 若用户未提供角色特征，系统根据台词内容推测角色语气，从语音库中随机匹配合适音色
     3. 不同角色自动分配不同音色，避免重复
   - 语音库为空时：弹窗引导用户去挑选/克隆声音

E. **非剧本内容处理**：
   - 若上传的txt文件无法被识别为剧本格式（无明确角色标识）
   - 系统将全文作为单一中性独白，使用默认音色朗读生成MP3

F. **异常处理**：

| 异常场景 | 处理方式 |
|----------|----------|
| 文件格式非txt | 前端限制 + 后端校验，提示"仅支持txt格式文件" |
| 文件大小超5MB | 提示"文件过大，请上传小于5MB的剧本文件" |
| 角色超过3个 | 弹窗提示"剧本角色过多，当前仅支持3个角色以内的小型剧本"，停止配音 |
| 单角色台词超过50句 | 截取前50句，提示"台词数量超限，已自动截取前50句" |
| 无法识别任何角色 | 按中性独白全文朗读 |
| 语音库为空 | 弹窗引导用户去挑选/克隆声音 |
| 合成过程中断 | 提示"配音生成失败，请重试" |

#### 4. 输出

| 输出项 | 描述 |
|--------|------|
| **输出格式** | MP3音频文件 |
| **输出内容** | 完整的多人配音音频（所有角色台词按剧本顺序拼接） |
| **音频参数** | 采样率≥22050Hz，比特率≥128kbps |
| **存储位置** | 对象存储（与TTS合成音频统一管理） |
| **处理时间** | 预计30秒-3分钟（取决于剧本长度），后台异步处理 |

**结果页面展示**：
- 音频播放器（在线试听）
- 下载按钮（下载MP3文件）
- 修改调整入口（可重新选择音色、调整参数后重新生成）
- 生成记录保存至合成历史

### 模块三：分享平台（音色模型分享）

#### 1. 功能介绍

分享平台模块允许用户将自己克隆训练好的音色模型发布到平台公共区，供其他用户浏览、试听和下载。下载后的音色可直接用于TTS合成和剧本智能配音。MVP阶段简化实现：无需审核、无需描述标签、无需下载限制。

**核心功能列表**：
1. 用户可将自己克隆的音色发布到公共音色区
2. 公共音色区支持浏览和关键词搜索
3. 其他用户可试听共享音色的效果
4. 其他用户可下载共享音色到自己的语音库
5. 分享者可随时取消分享/下架自己的音色
6. 下载的音色可用于TTS合成和剧本配音

#### 2. 输入

**音色分享（发布方）**：

| 输入项 | 描述 |
|--------|------|
| **输入来源** | 用户在音色管理页面选择已克隆的音色，点击「分享到平台」 |
| **前置条件** | 音色状态为「就绪」（克隆训练已完成） |
| **分享操作** | 一键分享，无需填写额外信息（MVP阶段） |

**音色浏览与下载（使用方）**：

| 输入项 | 描述 |
|--------|------|
| **搜索关键词** | 音色名称关键词（可选，字符串，最长50字） |
| **分页参数** | 页码、每页数量（默认12个） |
| **排序方式** | 按发布时间降序（默认） |

#### 3. 处理流程

**分享流程**：
```
用户进入音色管理页面
    ↓
选择已就绪的音色
    ↓
点击「分享到平台」按钮
    ↓
二次确认弹窗：「确定要将该音色分享到公共平台吗？分享后其他用户可下载使用。」
    ↓ （确认）
音色状态更新为「已分享」
    ↓
音色出现在公共音色区
```

**浏览与下载流程**：
```
用户进入「音色市场」页面
    ↓
浏览公共音色列表（分页展示）
    ↓
支持关键词搜索
    ↓
点击音色卡片 → 查看详情 + 试听
    ↓
点击「下载」按钮
    ↓
音色模型文件下载至用户语音库
    ↓
下载完成后即可在TTS合成/剧本配音中使用
```

**取消分享流程**：
```
用户进入音色管理页面
    ↓
选择已分享的音色
    ↓
点击「取消分享」按钮
    ↓
二次确认弹窗
    ↓ （确认）
音色从公共区移除，已下载用户不受影响
```

**处理规格**：

A. **分享规则**：
   - 仅「就绪」状态的音色可分享
   - 分享后音色原文件保留在分享者账号下
   - 分享操作不产生新的模型文件拷贝（公共区展示的是引用）
   - 取消分享后，已下载该音色的用户仍可继续使用

B. **公共音色区展示**：
   - 列表项展示：音色名称、分享者（用户名脱敏）、下载次数、发布时间
   - 点击可查看详情：音色名称、试听样本、下载次数、发布时间
   - 支持按音色名称关键词搜索
   - 支持分页（每页12个）

C. **下载处理**：
   - 下载操作将音色模型文件拷贝至下载者语音库
   - 下载的音色与用户克隆的音色在使用上无区别
   - 下载的音色同样支持在TTS合成和剧本配音中使用
   - 下载的音色不可再次分享（仅原始分享者可管理分享状态）

D. **异常处理**：

| 异常场景 | 处理方式 |
|----------|----------|
| 分享的音色正在训练中 | 按钮置灰，提示"音色训练完成后才能分享" |
| 下载时网络中断 | 提示"下载失败，请重试"，支持断点续传 |
| 分享者删除了原始音色 | 已分享的音色自动下架，已下载用户不受影响 |
| 音色模型文件损坏/丢失 | 提示"音色文件异常，无法下载" |

#### 4. 输出

**音色分享输出**：

| 输出项 | 描述 |
|--------|------|
| 分享状态 | 音色状态更新为「已分享」，出现在公共音色区 |
| 通知 | 分享成功后前端Toast提示 |

**音色浏览输出**：

| 输出项 | 描述 |
|--------|------|
| 公共音色列表 | JSON格式，包含音色ID、名称、分享者、下载次数、发布时间 |
| 分页信息 | 总条数、当前页、总页数 |
| 试听音频 | 音色预设文本的合成MP3（临时签名URL） |

**音色下载输出**：

| 输出项 | 描述 |
|--------|------|
| 下载结果 | 音色模型文件下载至用户语音库，下载成功后前端Toast提示 |
| 下载后状态 | 音色出现在用户的语音库中，状态为「已下载」，可用于合成 |

## 设计决策

### 技术方案

- **架构选择**：Web 应用（PC 端优先），前后端分离架构
  - 前端：React + TypeScript，响应式设计，后续可扩展至移动端适配
  - 后端：Python（FastAPI），便于集成 AI 模型推理
  - 数据库：PostgreSQL（用户/业务数据） + 对象存储（音频文件/音色模型文件）

- **关键技术优化**（基于架构评审）：
  1. **AI推理抽象层**：统一 `InferenceEngine` 接口，支持 GPT-SoVITS / CosyVoice / ONNX CPU 多引擎路由，故障切换分钟级，业务代码零改动
  2. **4级优先级任务队列**：TTS(P0/CPU) → 即时克隆(P1) → 深度克隆(P2) → 剧本配音(P3)，确保高频轻量任务不被重型任务阻塞
  3. **ONNX CPU兜底**：TTS合成模型导出为ONNX格式，GPU忙时自动降级到CPU推理，响应时间＜5秒
  4. **模块接口隔离**：5大核心模块（VoiceCloning、TTS、ScriptDub、Share、Notification）基于抽象接口依赖，预埋绞杀者模式，后期微服务拆分无需大规模重构

- **关键组件**：
  - 音频预处理服务：降噪、VAD（人声检测）、格式转换
  - AI推理抽象层：统一接口 `InferenceEngine(Protocol)`，注册路由自动选择最优引擎
  - 语音克隆引擎（GPU）：GPT-SoVITS（主推）/ CosyVoice（备选）
  - TTS推理服务（GPU/CPU）：GPU版 GPT-SoVITS + CPU版 ONNX Runtime
  - 优先级任务队列：Celery + Redis，4级优先级（P0-P3）
  - 通知服务：站内通知 + 短信通知（克隆完成）
  - 情感分析引擎：基于BERT的中文情感分类模型（bert-base-chinese微调）
  - 剧本解析引擎：基于规则 + NLP命名实体识别（正则匹配角色标识 + NER辅助）
  - 音频拼接工具：FFmpeg，用于多角色音频拼接为完整MP3

- **数据存储**：
  - 关系型数据：PostgreSQL 存储用户、音色元数据、合成记录、剧本任务、分享记录
  - 文件存储：MinIO / 阿里云 OSS 存储音频文件和音色模型
  - 缓存：Redis 缓存热门音色、会话信息

- **接口设计**：
  - RESTful API + WebSocket（克隆进度/配音进度实时推送）
  - 核心接口：用户认证、音频上传、克隆任务管理、TTS 合成、音色管理、剧本配音、音色分享

### 关键技术决策

| 决策点 | 方案 | 理由 |
|--------|------|------|
| AI推理抽象层 | `InferenceEngine(Protocol)` + 引擎注册路由 | 解耦业务代码与AI模型，支持一键切换备选模型，降低单模型风险 |
| TTS推理路径 | GPU首选（GPT-SoVITS）+ CPU兜底（ONNX Runtime） | GPU空闲时高精度推理，GPU忙时自动降级CPU，保证TTS秒级响应 |
| 任务调度策略 | Celery 4级优先级队列（P0-P3） | TTS(P0/CPU)永不排队，即时克隆(P1)优先于深度克隆(P2)，配音(P3)最低 |
| 模块间依赖 | 基于ABC抽象接口（绞杀者模式） | 单体阶段解耦模块，微服务阶段无需重构，仅需将package独立部署 |
| 情感分析模型 | 基于BERT的中文情感分类模型（bert-base-chinese微调） | 中文场景效果好，支持6分类，推理速度快，CPU可运行 |
| 角色分割方案 | 基于规则 + NLP命名实体识别（正则匹配角色标识 + NER辅助） | 剧本格式相对固定，规则为主即可覆盖MVP场景 |
| 音色匹配策略 | 基于用户语音库随机分配 + 角色去重 | MVP简化实现，后续可升级为特征匹配 |
| 音频拼接 | 逐句合成后使用FFmpeg拼接为完整MP3 | 技术成熟，可控性好，便于后期单句替换 |
| 分享存储 | 分享不产生物理拷贝，下载时复制模型文件 | 节省存储空间，分享者删除后已下载用户不受影响 |

### 约束条件

- **性能要求**：MVP 阶段不做硬性要求，优先保证功能可用
  - 参考目标：TTS 合成 ≤10 秒，即时克隆 ≤5 分钟，剧本配音 ≤3 分钟
- **兼容性**：
  - 浏览器：Chrome、Edge、Firefox 最新两个版本
  - 移动端浏览器：基础可用即可，不要求完美适配
- **安全性**：
  - 手机号加密存储
  - API 接口鉴权（JWT Token）
  - 文件上传类型和大小校验
- **可扩展性**：
  - 音色模型存储结构预留音色市场扩展（音色公开/私有标识）
  - 用户表预留会员等级字段
  - TTS 参数接口预留扩展位
- **基础设施依赖**：
  - 语音克隆依赖GPT-SoVITS/CosyVoice推理引擎，需GPU支持
  - TTS合成支持双路径：GPU走GPT-SoVITS（高精度），CPU走ONNX Runtime（轻量兜底）
  - 情感分析模型运行于CPU（约2-4GB内存），无需GPU
  - 分享平台依赖对象存储的拷贝能力
  - MVP阶段单GPU处理深度克隆和配音任务，TTS合成走CPU不占用GPU

### 风险评估（含优化措施后更新）

| 风险 | 等级 | 缓解措施 |
|------|:----:|----------|
| 开源模型（GPT-SoVITS/CosyVoice）的克隆效果稳定性 | 🟡 中↓ | **优化**：推理接口抽象化，GPT-SoVITS效果不达标时一键切换至CosyVoice，业务代码零改动；需提前benchmark验证 |
| GPU 资源成本 | 🟡 中 | **优化**：TTS合成走ONNX CPU路径，不占用GPU；深度克隆和配音使用单GPU串行处理，成本可控 |
| TTS合成响应延迟（新） | 🟢 低 | **优化**：ONNX CPU兜底 + P0优先级队列，TTS秒级返回不排队 |
| 并发任务排队（新） | 🟢 低 | **优化**：4级优先级队列（P0/P1/P2/P3），前端展示排队位置和预估时间 |
| 单体架构拆分成本（新） | 🟢 低 | **优化**：模块接口隔离 + 绞杀者模式，拆分时无需大规模重构 |
| 短信服务商稳定性 | 🟡 中 | 选择有SLA保障的服务商（如阿里云短信），准备备用通道 |
| 对象存储可用性 | 🟢 低 | 选择成熟云服务商，本地保留音频缓存 |
| 模型微调和部署调试周期不确定 | 🟡 中 | 预留2周模型调试buffer；预构建Docker GPU镜像减少环境配置时间 |
| 前后端联调时间可能超出预期 | 🟢 低↓ | **优化**：MockEngine提供完整模拟接口，前端/后端/AI可三线并行开发 |
| 情感分析准确率不足 | 🟡 中 | 预留手动调整入口，收集用户反馈持续优化模型 |
| 剧本角色分割失败率高 | 🟡 中 | 支持多种剧本格式，非剧本兜底为中性朗读 |
| 配音生成时间超预期 | 🟢 低 | 异步处理 + 进度通知，用户无需等待 |
| 分享音色质量参差不齐 | 🟢 低 | MVP不设审核，后续版本接入审核和评分机制 |

## 验收标准

### 模块一：语音克隆与TTS合成

- [ ] **AC-001**：用户可通过手机号+验证码完成注册和登录，验证码有效期 5 分钟
- [ ] **AC-002**：支持 MP3/WAV/M4A/OGG 格式上传，文件大小 ≤50MB，上传进度可视化
- [ ] **AC-003**：上传音频后 5 分钟内生成可试听的克隆音色（即时克隆）
- [ ] **AC-004**：提交深度克隆任务后，后台异步训练，完成后站内通知，音色可试听
- [ ] **AC-005**：克隆失败时展示明确失败原因，提供「重新克隆」入口
- [ ] **AC-006**：用户可查看、试听、删除自己的克隆音色
- [ ] **AC-007**：输入 ≤1000 字文本，选择音色，可调节语速/音量/音调，生成 MP3 音频
- [ ] **AC-008**：语速范围 0.5x-2.0x，音量 0-100%，音调 -12 ~ +12，调节后合成效果有明显变化
- [ ] **AC-009**：合成结果支持在线播放和 MP3 下载
- [ ] **AC-010**：展示历史合成记录，支持重新生成和下载

### 模块二：剧本智能配音

- [ ] **AC-011**：Given 用户上传包含3个角色的txt剧本 → When 点击"生成配音" → Then 系统自动完成配音并在3分钟内返回MP3音频
- [ ] **AC-012**：Given 生成的配音音频 → When 用户点击播放 → Then 可正常在线试听完整配音
- [ ] **AC-013**：Given 用户对生成结果不满意 → When 用户点击"修改"调整音色后重新生成 → Then 系统使用新音色重新合成
- [ ] **AC-014**：Given 用户上传非剧本格式的txt文件 → When 系统处理 → Then 按中性独白全文朗读生成MP3
- [ ] **AC-015**：Given 剧本包含4个角色 → When 系统识别 → Then 弹窗提示"剧本角色过多"，停止配音
- [ ] **AC-016**：Given 用户语音库为空 → When 用户尝试生成配音 → Then 弹窗引导用户去挑选/克隆声音
- [ ] **AC-017**：Given 台词情感为"悲伤" → When 系统合成 → Then 该句配音音调偏低、语速偏慢
- [ ] **AC-018**：Given 台词情感为"喜悦" → When 系统合成 → Then 该句配音音调偏高、语速偏快
- [ ] **AC-019**：Given 用户上传的txt编码为GBK → When 系统读取 → Then 自动识别编码并正确解析
- [ ] **AC-020**：Given 剧本中某句台词情感模糊 → When 系统分析 → Then 默认使用中性情感参数朗读

### 模块三：分享平台

- [ ] **AC-021**：Given 用户有一个已就绪的克隆音色 → When 用户点击"分享到平台"并确认 → Then 音色出现在公共音色区
- [ ] **AC-022**：Given 公共音色区有多个分享音色 → When 用户进入"音色市场"页面 → Then 分页展示所有已分享音色
- [ ] **AC-023**：Given 用户在音色市场搜索"温柔" → When 用户点击搜索 → Then 列表筛选出名称包含"温柔"的音色
- [ ] **AC-024**：Given 用户点击某个分享音色的"下载" → When 下载完成 → Then 该音色出现在用户语音库中
- [ ] **AC-025**：Given 用户下载了一个分享音色 → When 用户进入TTS合成 → Then 该音色可选用于合成
- [ ] **AC-026**：Given 分享者点击"取消分享"并确认 → When 操作完成 → Then 音色从公共区移除，已下载用户不受影响
- [ ] **AC-027**：Given 音色正在训练中 → When 用户尝试分享 → Then 按钮置灰，提示"训练完成后才能分享"
- [ ] **AC-028**：Given 分享者删除了已分享的原始音色 → When 其他用户访问 → Then 该分享自动下架
