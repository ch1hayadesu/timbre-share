## 数据库部署顺序

```powershell
# 1. 创建数据库及表结构
mysql -u root timbre_share --default-character-set=utf8mb4 -e "source sql/01_ddl.sql"

# 2. 导入种子数据
mysql -u root timbre_share --default-character-set=utf8mb4 -e "source sql/02_seed_data.sql"

# 3. 创建用户并授权
mysql -u root timbre_share --default-character-set=utf8mb4 -e "source sql/03_create_users.sql"

# 4. 导入测试数据（可选，仅开发/测试环境）
mysql -u root timbre_share --default-character-set=utf8mb4 -e "source sql/04_test_data.sql"
```

## 数据备份

```powershell
# 全量备份
& "C:\Program Files\MySQL\MySQL Server 8.4\bin\mysqldump.exe" -u root --single-transaction --routines --triggers --events timbre_share > backup_$(Get-Date -Format yyyyMMdd).sql

# 仅结构
& "C:\Program Files\MySQL\MySQL Server 8.4\bin\mysqldump.exe" -u root --no-data timbre_share > schema_only.sql

# 仅数据
& "C:\Program Files\MySQL\MySQL Server 8.4\bin\mysqldump.exe" -u root --no-create-info timbre_share > data_only.sql
```

## 数据恢复

```powershell

```

## 用户账户

| 用户名 | 用途 | 连接来源 | 权限范围 |
|--------|------|----------|----------|
| `app_user` | Java Spring Boot 应用 | 127.0.0.1, localhost | timbre_share: SELECT, INSERT, UPDATE, DELETE |
| `readonly_user` | 报表/监控 | % (任意), 127.0.0.1, localhost | timbre_share: SELECT |
| `admin_user` | DBA 运维 | 127.0.0.1, localhost | timbre_share: ALL PRIVILEGES WITH GRANT OPTION |

> **安全提醒**：密码存储在 `sql/03_create_users.sql`，首次部署后立即修改：
> ```sql
> ALTER USER 'app_user'@'127.0.0.1' IDENTIFIED BY '新密码';
> ALTER USER 'readonly_user'@'127.0.0.1' IDENTIFIED BY '新密码';
> ALTER USER 'admin_user'@'127.0.0.1' IDENTIFIED BY '新密码';
> ```

## 快速诊断

```sql
-- 查看当前连接
SELECT * FROM information_schema.PROCESSLIST;

-- 查看表空间大小
SELECT table_name, ROUND(((data_length + index_length) / 1024 / 1024), 2) AS size_mb
FROM information_schema.tables WHERE table_schema = 'timbre_share';

-- 查看最近创建的10条分享
SELECT * FROM voice_share ORDER BY created_at DESC LIMIT 10;

-- 检查失败的TTS任务
SELECT * FROM tts_record WHERE status = 2;
```
